One quick note about cloning GitHub projects from the links provided throughout this course. If you wish to make commits and push code up after cloning down from any of the GitHub links, you should actually fork the project first. The reason for this is because the GitHub project links provided are pointing to repositories you do not own, in order to own the repository, you have to fork it so you get a copy of the project in your own GitHub account that you do own!


You can see the fork button in the top right corner of every GitHub project; click it and a copy of the project will be added to your GitHub profile under the same name as the original project.



After forking the project, simply clone it the way you would from the new forked project in your own GitHub repository and you can commit and push to it freely!
